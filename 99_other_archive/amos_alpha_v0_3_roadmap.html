v0.3A

Manual Data Alpha

Make AMOS usable with manually entered daily data for priority tickers.

Fastest path to usefulness without waiting for API automation.

Deliverables

  • Manual data input sheet/schema
  • Priority ticker field checklist
  • Daily Brief generated from manually updated score packet
  • CRDO/NVDA/LITE/HOOD/IREN manually refreshed pages
  • Manual evidence packet template

Success Criteria

  • User can update 5-10 fields per ticker and get a coherent Daily Brief.
  • AMOS can distinguish Hold / Wait / Trim Watch / Build Watch from manual inputs.
  • Every action output includes evidence, counter-evidence and invalidation.
v0.3B

Semi-automated Data Alpha

Allow CSV/JSON imports from charting/broker/research tools.

Reduce manual burden while keeping control and transparency.

Deliverables

  • CSV import schema
  • Broker/export parser
  • TradingView-style technical snapshot parser
  • Manual + imported data conflict checker
  • Data freshness warning system

Success Criteria

  • System flags stale data automatically.
  • Imported technical data updates stock pages and dashboard.
  • Conflicting manual vs imported values are visible, not hidden.
v0.3C

Real-time / API-ready Alpha

Prepare AMOS to connect to live market/fundamental/options/event APIs.

Daily use needs freshness, but API data must enter through controlled adapters.

Deliverables

  • Market data adapter
  • Macro data adapter
  • Earnings/fundamental adapter
  • Options/gamma adapter
  • Event calendar adapter
  • Evidence/source adapter

Success Criteria

  • Daily Brief can regenerate from fresh data packets.
  • Stock pages update key scores automatically.
  • Decision confidence is capped when required fields are stale/missing.
v0.3D

Review & Learning Alpha

Build prediction-action-result review loops.

AMOS should improve by testing which signals worked for which stock personality.

Deliverables

  • Prediction log
  • Action log
  • Result review
  • Signal effectiveness by ticker
  • Source quality update
  • Behavior guardrail update

Success Criteria

  • Each major decision can be reviewed later.
  • System learns which indicators work better for CRDO vs HOOD vs IREN etc.
  • User behavior risk is tracked without moralizing or generic warnings.

Engineering Guardrails

工程护栏

  • Do not expand new modules before v0.3A manual workflow is usable.
  • Do not pretend sample data is live data.
  • Do not output action without invalidation conditions.
  • Do not let Gann alone trigger buy/sell.
  • Do not let user observation replace market evidence.
