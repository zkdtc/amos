Closed feedback loops unique to NVDA

The diagram exposes three NVDA-specific closed loops that don't exist for any other ticker in AMOS:

  1. NVDA → Macro Regime → NVDA-the-stock. NVDA's own price feeds the leadership axis of the macro regime. The regime then back-caps every per-ticker action — including NVDA itself. This is the "leader is the regime barometer" pattern from single-jianwei (龙头是流动性聚焦点).

  2. NVDA → other tickers' Personality.LeadershipDependency. Every non-NVDA ticker uses NVDA's bars as the leader benchmark for the 5th radar axis. So NVDA's behavior literally shapes how every other stock is described.

  3. NVDA earnings → MacroEvent.Critical → action cap on NVDA. The earnings date pulled from quoteSummary becomes a Critical event, which sets aggregateEventCap = No Attack on NVDA's own page through the strictest-wins rule.

AMOS is research-only. No brokerage connection. Gann never triggers trades alone. Counter-evidence required for any claim to influence action. Anchor Verification gate required before formal Gann use. Critical events default to No Attack.

Why this is named "NVDA Metric Dependency Graph"

NVDA is unique in AMOS because it appears in three different roles simultaneously:

  1. As a tracked ticker — its own page.

  2. As a benchmark — fed into the macro regime alongside SPY/QQQ/VIX/etc.

  3. As the leader reference for every other ticker's Personality Radar (axis 5).

The diagram therefore has more incoming + outgoing edges for NVDA than for any other ticker. Rendering it explicitly makes the closed loops visible — which is what single-jianwei calls 龙头反身性 (leader reflexivity).

Decision-making rules (purely rule-based · no ML · fully deterministic)

AMOS makes no ML predictions, runs no LLM in the decision path, and never emits "buy / sell" signals. It computes an Effective Action Cap — the strictest ceiling on what is operationally allowed today — by running six independent classifiers and taking the most restrictive output. The whole pipeline is deterministic TypeScript that fits in a single screen.

2 · Action Cap priority ladder

The composer picks the strictest (most restrictive) cap across all classifiers — there is no weighted scoring, no fuzzy combination.

#CapMeans
1 (least)AttackAll gates open. Aggressive position changes permitted.
2NormalStandard playbook.
3Reduced SizeTactical only; no aggression.
4WatchObserve; no execution this session.
5Research OnlyInformation OK; no operational decisions.
6 (most)No AttackDefault-deny. Critical event window or Risk-Off regime.

3 · The six classifiers

#ClassifierRuleModule
1FreshnessFresh → Normal · Stale → Reduced Size · Missing → Research Only. Freshness is market-aware: outside session hours the last close stays Fresh until the next open.rules/actionCaps.ts
2Gann data completenessengine.hard_missing_data.length > 0 → Research Only, else Normal. Gann quality never relaxes; it can only restrict.rules/actionCaps.ts
3Event proximity (7-day window)For every event ahead: Critical → No Attack, High → Reduced Size, Medium → Watch, Low → Normal. Strictest across all events wins.rules/eventGuards.ts
4Anchor Verification gateIf any Gann anchor for the ticker is usePermission !== 'Verified' → Research Only. Hard gate. A human must stamp Verified.StockIntelligencePage.tsx + AnchorVerificationGate.tsx
5Macro RegimeRisk-Off → No Attack · Defensive → Reduced Size · otherwise no contribution. Regime is itself computed by a 3-axis state machine (see below).data/marketRegime.ts
6Evidence / Counter-EvidenceA claim missing counterEvidence is Blocked and cannot influence the Decision Summary. Doesn't change the cap directly but suppresses bullish/bearish inputs.rules/evidenceGuards.ts

5 · Strictest-wins composition (the actual code)

// amos/src/pages/StockIntelligencePage.tsx (excerpt)
const caps: ActionCap[] = [];
caps.push(freshnessCap(input.freshness));                                 // 1
caps.push(gannMissingDataCap(engine));                                    // 2
caps.push(aggregateEventCap(events));                                     // 3
if (!anchors.every(a => a.usePermission === 'Verified'))
  caps.push('Research Only');                                             // 4
if (regime.label === 'Risk-Off')   caps.push('No Attack');                // 5
if (regime.label === 'Defensive')  caps.push('Reduced Size');             // 5
// evidence guard runs on a per-claim basis in EvidencePanel              // 6
const cap = strictestCap(caps);   // pure min() over priority table

8 · What is NOT used to decide

  • No ML. Nothing fits, predicts, or learns.
  • No LLM. Decision path is plain deterministic TypeScript.
  • No price targets. The cap is a ceiling on aggression, never a directional call.
  • Gann scores, RSI, Personality axes, Valuation zones, Narrative phase, Options gamma, Peers — all displayed for human reasoning but do not change the cap. Single-jianwei doctrine: "Gann cannot trigger trades alone."

9 · Where a human is required (three hard gates)

  1. Anchor Verification — a real human must stamp anchors before Gann context unlocks.
  2. Counter-Evidence — a claim is Blocked until "what would prove it wrong" is filled in.
  3. Post-event Review — events flagged postEventReviewRequired: true force a manual checkpoint before caps relax.
